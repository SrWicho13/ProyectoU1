Introducción

El proyecto "Gestión de Tareas" es una aplicación de escritorio desarrollada con NodeJS express con el objetivo de facilitar la administración de tareas personales o de equipo. La aplicación permite a los usuarios crear, visualizar, modificar y eliminar tareas de manera eficiente. El sistema ofrece una experiencia fluida para la gestión de actividades diarias. Esta documentación proporciona una guía completa para instalar, configurar y utilizar el sistema, junto con una referencia técnica de las funcionalidades y clases principales.

Requisitos

Para ejecutar correctamente este proyecto de gestión de tareas, asegúrate de cumplir con los siguientes requisitos:

  • Node.js versión 16 o superior
  • MongoDB como base de datos para almacenar las tareas
  • Dependencias adicionales gestionadas mediante npm, incluyendo:
    • bcrypt: para el manejo seguro de contraseñas
    • express: framework para manejar las solicitudes HTTP
    • firebase-admin: para integración con Firebase, si es necesario
    • jsonwebtoken: para autenticación basada en tokens JWT
    • mongoose: para el manejo de datos en MongoDB
  • Para pruebas y desarrollo, asegúrate de instalar también las siguientes dependencias de desarrollo:
    • chai y mocha: para pruebas unitarias
    • supertest: para pruebas de API

Instalación

Para instalar y configurar el proyecto, sigue estos pasos:

  1. Clona el repositorio: Clona el repositorio en tu máquina local utilizando git clone [URL_DEL_REPOSITORIO].
  2. Instala las dependencias: Navega a la carpeta raíz del proyecto y ejecuta npm install para instalar todas las dependencias listadas en package.json.
  3. Configura el archivo .env: Crea un archivo .env en la raíz del proyecto y define las variables de entorno necesarias (por ejemplo, conexión a MongoDB, claves JWT, etc.).
  4. Inicia la aplicación: Ejecuta npm start para iniciar el servidor o npm run dev si estás usando nodemon para desarrollo.

Documentación Técnica

Esta sección describe las principales funcionalidades y controladores del proyecto:

Controladores Principales

TareasController.js

Controlador que maneja todas las operaciones CRUD para las tareas. Incluye métodos para crear, actualizar, obtener y eliminar tareas en la base de datos.

AuthController.js

Controlador encargado de la autenticación de usuarios. Utiliza jsonwebtoken para generar y verificar tokens JWT, asegurando que sólo los usuarios autenticados puedan acceder a ciertos endpoints.

db.js

Módulo de conexión a MongoDB utilizando mongoose. Configura la conexión y gestiona cualquier error de conectividad.

Endpoints y Funcionalidades

El sistema permite realizar operaciones CRUD sobre las tareas a través de los siguientes endpoints:

  • POST /api/tareas: Crea una nueva tarea. Requiere autenticación y un cuerpo JSON con los detalles de la tarea.
  • GET /api/tareas: Recupera todas las tareas disponibles. Requiere autenticación.
  • PUT /api/tareas/:id: Actualiza una tarea específica identificada por su ID. Requiere autenticación.
  • DELETE /api/tareas/:id: Elimina una tarea específica. Requiere autenticación.

Ejemplos de Uso

Crear una nueva tarea

Envía una solicitud POST a /tasks con el siguiente cuerpo JSON para crear una nueva tarea:


    POST /tasks
    Headers:
    Authorization: //Token de autenticación
    x-api-key: //Clave de API
    
    Body:
    {
        "title": "Tarea de prueba",
        "description": "Esta es una tarea de prueba"
    }

Obtener todas las tareas

Envía una solicitud GET a /tasks para obtener todas las tareas del usuario autenticado:


    GET /tasks
    Headers:
    Authorization: //Token de autenticación
    x-api-key: //Clave de API
    
    Response:
    [
        {
            "id": "1",
            "title": "Tarea de prueba",
            "description": "Esta es una tarea de prueba",
            "completed": false
        },
        ...
    ]

Actualizar una tarea

Envía una solicitud PUT a /tasks/:id con el ID de la tarea en la URL para actualizar los detalles de una tarea existente:


    PUT /tasks/1
    Headers:
    Authorization: //Token de autenticación
    x-api-key: //Clave de API
    
    Body:
    {
        "title": "Tarea actualizada",
        "description": "Descripción actualizada",
        "completed": true
    }
    
    Response:
    {
        "id": "1",
        "title": "Tarea actualizada",
        "description": "Descripción actualizada",
        "completed": true
    }

Eliminar una tarea

Envía una solicitud DELETE a /tasks/:id con el ID de la tarea en la URL para eliminar una tarea existente:


    DELETE /tasks/1
    Headers:
    Authorization: //Token de autenticación
    x-api-key: //Clave de API
    
    Response:
    {
        "message": "Tarea eliminada correctamente"
    }
